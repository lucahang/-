# High-Performance Windows Remote Desktop System (C++)

本项目是一款基于 C++ 开发的**超低延迟、高性能 Windows 远程控制系统**。系统采用经典的 C/S（客户端/服务端）架构，针对实时视频流传输进行了深度优化。通过双通道网络架构（TCP 信令/UDP 视频）与硬件全链路加速，成功在局域网/公网穿透环境下实现了 **1080P 60FPS** 的丝滑远控体验。

---

## 🚀 核心架构与技术亮点

系统全面摒弃了传统的 GDI 截屏与 CPU 编码方案，构建了一条**从采集、编码到传输、解码、渲染的全链路硬件加速管线**：

### 1. 全链路硬件加速架构 (Hardware-Accelerated Pipeline)
*   **底层画面采集：** 深入 Windows 内核，采用 **DXGI (Desktop Duplication API)** 实现了高性能屏幕捕获，完美支持 **1080P 60FPS** 的高帧率画面采集（代码中已预留并封装 `ID3D11Texture2D` 直接捕获接口，具备极强的 GPU 联动扩展性）。
*   **GPU 硬件编码：** 集成 NVIDIA 硬件加速接口 **NVENC**，利用 GPU 独立多媒体引擎进行 H.264 实时编码。开启 **`zerolatency`（零延迟预设）**，大幅压低编码耗时与 CPU 负载。
*   **GPU 硬件解码与渲染：** 客户端采用 **GPU 硬件解码**，并直接基于 **Direct3D (D3D)** 进行图像渲染加速，避免了 CPU 软解码和内存拷贝（零拷贝思想），实现了画面无缝输出。

### 2. 双通道网络架构与私有协议设计 (Dual-Channel Network)
*   **信令/视频分离：** 系统采用双通信通道设计。鼠标、键盘控制指令以及心跳包通过 **TCP 通道** 传输，确保控制精准、不丢包；视频码流通过 **UDP 通道** 传输，保证极高的实时性。
*   **自定义应用层协议：** 自研流媒体私有报头，报文格式包含：`包长度 (Length)`、`帧序号 (Frame ID)`、`分片索引 (Slice Index)`，支持大视频帧的动态切片与组包。
*   **自适应丢包策略（抗抖动）：** 针对 UDP 弱网环境，设计了主动丢弃策略。当检测到当前帧存在丢包或接收到更新的帧序号时，主动放弃历史过期分片，直接平滑过渡至最新帧，有效防止了画面堆积和长延时。
*   **多线程并发控制：** 客户端采用高效的**双线程异步架构**，将“网络数据接收”与“视频流硬解码”完全解耦，互不阻塞，极大提升了吞吐量。

### 3. 网络拓扑与穿透能力 (Network & Penetration)
*   系统原生支持局域网高带宽传输。
*   支持通过阿里云等公网服务器部署 **FRP (Fast Reverse Proxy)** 架构，轻松实现安全的**外网内网穿透**，具备跨地域远程控制能力。

## 🛠️ 技术栈清单 (Tech Stack)

*   **开发语言：** 现代 C++ (C++11/14)
*   **多媒体与图形：** FFmpeg, NVENC (NVIDIA GPU 编码), DXGI API, Direct3D (客户端硬件渲染)
*   **网络编程：** 高性能 WinSock 编程, TCP/UDP 双协议通道, 自定义应用层流媒体协议
*   **内网穿透：** FRP, Alibaba Cloud (ECS)
